# Сайт Nazar Kokoten — як це працює

## Структура

- `*.html` + `style.css` + `script.js` — сам сайт (верстка, її не чіпаєш)
- `content/*.json` — **весь контент**: тексти, ціни, проєкти, картинки
- `images/` — файли зображень
- `.pages.yml` — конфіг редактора Pages CMS (описує, які поля показувати)

Сторінки самі підтягують контент із `content/*.json` при завантаженні.
Через це сайт треба відкривати через сервер, а не подвійним кліком по файлу:
локально — `npx serve` у папці сайту, або просто дивитись на Vercel.

## Підключення (один раз, ~10 хвилин)

1. **GitHub.** Створи репозиторій (напр. `nazar-site`) і залий у нього всю цю папку.
   Можна без термінала: на сторінці репозиторію → Add file → Upload files → перетягни все.
   Важливо: файл `.pages.yml` має потрапити в корінь репозиторію.

2. **Vercel.** vercel.com → Add New Project → Import цього репозиторію → Deploy.
   Framework: Other, нічого не налаштовуєш. Отримуєш URL, пізніше підключиш свій домен.

3. **Pages CMS.** app.pagescms.org → Sign in with GitHub → дозволь доступ до репозиторію →
   відкрий його. Побачиш у сайдбарі секції українською: Загальне, Слайдер робіт,
   Головна, Про мене, Проєкти, Послуги та прайс, Контакт, Кейс, Футер.

## Щоденна робота

- Заходиш на app.pagescms.org (працює і з телефона)
- Редагуєш будь-яку секцію → Save
- Save = коміт у GitHub = Vercel сам передеплоює сайт за ~30–60 секунд. Все.

**Картинки:** у будь-якому полі зображення тисни на нього → завантаж файл —
він сам ляже в папку `images/` репозиторію. Є медіатека всіх завантажених файлів.

**Абзаци:** у великих текстових полях (Про мене, секції кейсу) порожній рядок = новий абзац.

**Кейс:** права колонка кейсу — це список блоків «Текстова секція» та «Зображення».
Їх можна додавати, видаляти й перетягувати у будь-якому порядку.
